概要 — What is ARAP?

As-Rigid-As-Possible (ARAP) は、メッシュ変形において「できる限り剛体変形に近い形状変化」を実現するアルゴリズムです。 ユーザーがいくつかの頂点（ハンドル）を移動させると、残りの頂点が局所的な剛体性を保ちながら自然に変形します。

O(n²)
疎線形システム（1回の解法）
SVD
各セルの最適回転行列推定
交互最適化
Local Step ↔ Global Step

アルゴリズムは Local-Global 交互最適化 を繰り返します：

Local Step（局所） — 各セルの局所座標を固定して、最適な回転行列 $R_i$ を SVD で求める
Global Step（大域） — 回転行列 $R_i$ を固定して、全頂点の新位置を線形システムで一括更新

STEP 1 エネルギー定式化

ARAP の目的関数（エネルギー）を最小化することで変形を求めます。 各頂点 $i$ の近傍 $\mathcal{N}(i)$ に対して、変形前後のエッジが回転行列 $R_i$ だけ異なるようにします。

ARAP エネルギー（論文 Eq. 7）
$$E(\mathbf{p}', \{R_i\}) = \sum_{i=1}^{n} w_i \sum_{j \in \mathcal{N}(i)} w_{ij} \left\| (\mathbf{p}'_i - \mathbf{p}'_j) - R_i(\mathbf{p}_i - \mathbf{p}_j) \right\|^2$$

記号の説明

記号意味
$\mathbf{p}_i$変形前の頂点 $i$ の座標（レスト形状）
$\mathbf{p}'_i$変形後の頂点 $i$ の座標（求める変数）
$R_i$頂点 $i$ に割り当てた最適回転行列（$3\times3$）
$w_{ij}$コタンジェント重み $= \frac{1}{2}(\cot\alpha_{ij} + \cot\beta_{ij})$
$\mathcal{N}(i)$頂点 $i$ に隣接する頂点の集合（1-ring 近傍）
コタンジェント重みは、三角形の角度が鋭いほど大きな重みを与えます。これにより、各エッジの「重要度」が幾何学的に意味のある形で組み込まれます。

コタンジェント重みの計算

コタンジェント重み
$$w_{ij} = \frac{1}{2}\left(\cot\alpha_{ij} + \cot\beta_{ij}\right)$$
$\alpha_{ij}$, $\beta_{ij}$ はエッジ $(i,j)$ を挟む2つの三角形における対角の角度 
// コタンジェント重みの計算
function cotangentWeight(pi, pj, pk) {
  // pk はエッジ(i,j) の対頂点
  const u = sub(pi, pk);  // ベクトル pk→pi
  const v = sub(pj, pk);  // ベクトル pk→pj
  const cosA = dot(u, v);
  const sinA = length(cross(u, v));
  return cosA / sinA;  // cot = cos/sin
}

function buildWeights(mesh) {
  const W = new Map();
  for (const [i, j, k] of mesh.triangles) {
    // 各エッジに対して2つの三角形からのcotを加算
    const wij = cotangentWeight(mesh.p[i], mesh.p[j], mesh.p[k]);
    addWeight(W, i, j, 0.5 * wij);
    addWeight(W, j, k, 0.5 * cotangentWeight(mesh.p[j], mesh.p[k], mesh.p[i]));
    addWeight(W, k, i, 0.5 * cotangentWeight(mesh.p[k], mesh.p[i], mesh.p[j]));
  }
  return W;
}

STEP 2 回転の推定 — Local Step (SVD)

Global Step で更新した頂点位置 $\mathbf{p}'$ を固定して、各頂点 $i$ の最適回転 $R_i$ を求めます。 これは各 $R_i$ を独立に解けるため、頂点ごとに並列計算できます。

共分散行列 $S_i$ の構築

頂点 $i$ のセル（1-ring 近傍）に対して共分散行列を組み立てます：

共分散行列（論文 Eq. 5）
$$S_i = \sum_{j \in \mathcal{N}(i)} w_{ij}\, \mathbf{e}_{ij}\, \mathbf{e}'^{\top}_{ij}$$
$\mathbf{e}_{ij} = \mathbf{p}_i - \mathbf{p}_j$（変形前エッジ），　$\mathbf{e}'_{ij} = \mathbf{p}'_i - \mathbf{p}'_j$（変形後エッジ）

SVD による最適回転の抽出

SVD 分解と回転行列の計算（論文 Eq. 6）
$$S_i = U \Sigma V^\top \quad \Rightarrow \quad R_i = V U^\top$$
$\det(VU^\top) = -1$ になる場合（反転）、$U$ の最小特異値に対応する列を符号反転して正しい回転行列に修正します。これは反射を回転に変換する処理です。 
// Local Step: SVD で各セルの最適回転を求める
function localStep(mesh, pNew) {
  const rotations = [];

  for (let i = 0; i < mesh.n; i++) {
    // 3×3 共分散行列 S_i を初期化（列優先）
    const S = zeros(3, 3);

    for (const j of mesh.neighbors[i]) {
      const w = mesh.W[i][j];
      // 変形前エッジ e_ij = p_i - p_j
      const e  = sub(mesh.p[i],  mesh.p[j]);
      // 変形後エッジ e'_ij = p'_i - p'_j
      const eP = sub(pNew[i], pNew[j]);

      // 外積（rank-1 行列）の重み付き和: S += w * e * eP^T
      for (let r = 0; r < 3; r++)
        for (let c = 3; c < 3; c++)
          S[r][c] += w * e[r] * eP[c];
    }

    // SVD: S = U Σ V^T
    const { U, V } = svd3x3(S);

    // 最適回転: R = V U^T
    let R = matMul(V, transpose(U));

    // 反射チェック: det(R) < 0 なら最小特異値列を反転
    if (det3(R) < 0) {
      U[0][2] *= -1; U[1][2] *= -1; U[2][2] *= -1;
      R = matMul(V, transpose(U));
    }
    rotations.push(R);
  }
  return rotations;
}

STEP 3 形状フィッティング — Global Step (線形システム)

Local Step で求めた回転行列 $\{R_i\}$ を固定して、全頂点の新位置 $\mathbf{p}'$ を求めます。 エネルギーを $\mathbf{p}'$ で微分してゼロとおくと、線形方程式系が得られます。

Global Step 方程式（論文 Eq. 8）
$$\sum_{j \in \mathcal{N}(i)} w_{ij}(\mathbf{p}'_i - \mathbf{p}'_j) = \sum_{j \in \mathcal{N}(i)} \frac{w_{ij}}{2}(R_i + R_j)(\mathbf{p}_i - \mathbf{p}_j)$$

これを行列形式で書くと：

行列形式（ラプラシアン系）
$$L\, \mathbf{p}' = \mathbf{b}$$
ラプラシアン行列 $L$： $$L_{ii} = \sum_{j \in \mathcal{N}(i)} w_{ij}, \quad L_{ij} = -w_{ij} \text{ (隣接)}, \quad L_{ij} = 0 \text{ (非隣接)}$$
右辺 $\mathbf{b}$ の $i$ 行目： $$\mathbf{b}_i = \sum_{j \in \mathcal{N}(i)} \frac{w_{ij}}{2}(R_i + R_j)\mathbf{e}_{ij}$$

ハンドル（固定頂点）の扱い

ユーザーが指定した固定頂点（ハンドル）の行は、対角成分を $1$、他を $0$ に置き換え、右辺を拘束座標にします（Dirichlet 境界条件）。 

// Global Step: ラプラシアン線形システムを構築して解く
function globalStep(mesh, rotations, handles) {
  const n = mesh.n;

  // ラプラシアン行列 L（疎行列）を構築
  const L = sparseMat(n, n);
  for (let i = 0; i < n; i++) {
    let sumW = 0;
    for (const j of mesh.neighbors[i]) {
      const w = mesh.W[i][j];
      L.set(i, j, -w);   // 非対角成分
      sumW += w;
    }
    L.set(i, i, sumW);   // 対角成分 = 行の重みの和
  }

  // 右辺ベクトル b を構築（x, y, z 各成分独立）
  const b = zeros(n, 3);
  for (let i = 0; i < n; i++) {
    for (const j of mesh.neighbors[i]) {
      const w  = mesh.W[i][j];
      const Ri = rotations[i];
      const Rj = rotations[j];
      const eij = sub(mesh.p[i], mesh.p[j]);  // 変形前エッジ
      // b_i += (w/2) * (Ri + Rj) * e_ij
      const rotated = vecAdd(
        matVec(Ri, eij),
        matVec(Rj, eij)
      );
      b[i] = vecAdd(b[i], scale(rotated, w * 0.5));
    }
  }

  // ハンドル（固定頂点）の行を上書き: L[h,:] = e_h, b[h] = handle_pos
  for (const [h, pos] of handles) {
    L.clearRow(h);
    L.set(h, h, 1);
    b[h] = pos;
  }

  // 疎線形システムを解く（Cholesky 分解 or 共役勾配法）
  return solveLinear(L, b);
}

STEP 4 反復収束

Local Step と Global Step を交互に繰り返すことでエネルギーが単調減少し収束します。 実用上は 3〜5回の反復で十分な精度が得られます。

アルゴリズム全体の流れ
入力: レスト形状 $\mathbf{p}$, ハンドル位置 $\mathbf{p}'_\text{handle}$
初期化: $\mathbf{p}' \leftarrow \mathbf{p}$（または前フレームの結果）
事前計算: コタンジェント重み $w_{ij}$, ラプラシアン行列 $L$（固定）

repeat $k = 1, 2, \ldots, K$ do
  // Local Step
  $\forall i$: $S_i \leftarrow \sum_j w_{ij}\,\mathbf{e}_{ij}\,\mathbf{e}'^{\top}_{ij}$
  $\forall i$: $R_i \leftarrow V_i U_i^\top$ (SVD of $S_i$)
  // Global Step
  $\forall i$: $\mathbf{b}_i \leftarrow \sum_j \tfrac{w_{ij}}{2}(R_i + R_j)\mathbf{e}_{ij}$
  $\mathbf{p}' \leftarrow L^{-1}\,\mathbf{b}$ (ハンドル拘束付き)
until 収束 or $k = K$

計算量の分析

ステップ計算量備考
重み事前計算$O(|F|)$フレーム数 $|F|$, 1回のみ
$L$ の因数分解（Cholesky）$O(n^{1.5})$ (疎)1回のみ（ハンドル変更時に再計算）
Local Step（SVD × $n$）$O(n)$各反復, $3\times3$ SVD は定数時間
$\mathbf{b}$ の構築$O(n)$各反復
$L^{-1}\mathbf{b}$（前向き/後退代入）$O(n)$各反復（因数分解済み）
ラプラシアン行列 $L$ はハンドル頂点が変わらない限り不変です。そのため Cholesky 因数分解を事前計算しておけば、各反復のコストは $O(n)$ に抑えられます。 
// 完全な ARAP ループ
function arap(mesh, handles, K = 5) {
  // 事前計算（ハンドルが変わらなければ再利用）
  const W = buildWeights(mesh);
  const L = buildLaplacian(W, handles);
  const chol = choleskyFactor(L);  // 事前因数分解

  let pNew = mesh.p.slice();  // 初期値 = レスト形状

  for (let k = 0; k < K; k++) {
    // --- Local Step ---
    const R = localStep(mesh, pNew);

    // --- Global Step ---
    const b = buildRHS(mesh, R);
    pNew = choleskyBackSolve(chol, b);  // L^{-1} b
  }

  return pNew;
}
